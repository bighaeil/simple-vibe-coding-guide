#### 3. 단계별 요청
```
✅ 
1단계: "OrderService 인터페이스를 만들어줘"
2단계: "이제 OrderServiceImpl을 구현해줘"
3단계: "OrderController를 만들어줘"
4단계: "단위 테스트를 작성해줘"

❌ "주문 시스템 전체를 한 번에 만들어줘"
```

#### 4. 점진적 개선
```
✅
1차: "기본 CRUD 메서드를 만들어줘"
2차: "여기에 페이징을 추가해줘"
3차: "검색 필터도 추가해줘"
4차: "성능 최적화해줘"

❌ "완벽한 API를 만들어줘"
```

### DON'T: 피해야 할 것 ❌

#### 1. 막연한 요청
```
❌ "좋은 코드 작성해줘"
❌ "최적화해줘"
❌ "잘 만들어줘"
```

#### 2. 검증 없이 사용
```
❌ AI가 생성한 코드를 바로 커밋
❌ 테스트 없이 배포
❌ 보안 검증 생략
```

#### 3. 모든 것을 AI에게 의존
```
❌ 비즈니스 로직 설계를 AI에게 맡김
❌ 아키텍처 결정을 AI에게 맡김
❌ 보안 취약점을 AI가 다 잡을 것이라 기대
```
